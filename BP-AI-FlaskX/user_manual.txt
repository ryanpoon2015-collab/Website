# BP-AI Health Monitoring System - User Manual

## 1. Introduction
The BP-AI Health Monitoring System is an integrated platform for tracking vital signs, including body temperature, blood pressure, pulse rate, and oxygen saturation (SpO2). It generates comprehensive AI-powered health reports and provides local and cloud backups of your data.

## 2. Getting Started
To start using the system:
1.  **Prepare the Devices**: Ensure the Arduino-based sensors (Temperature, Blood Pressure) and the Contec CMS60D Pulse Oximeter are connected to the USB ports of the computer.
2.  **Launch the Application**: Run the `app.py` script to start the Flask server.
3.  **Access the Dashboard**: Open your web browser and navigate to the local server address (typically `http://127.0.0.1:5000`).

## 3. Measuring Vitals

### Temperature
*   Click "Measure Temperature".
*   The system will trigger the Arduino sensor. 
*   Wait for approximately 7 seconds for the reading to stabilize and display.

### Blood Pressure
*   Click "Start BP Measurement".
*   The cuff will begin to inflate. Ensure you are sitting still and relaxed.
*   Once finished, the system will capture a picture of the BP monitor and extract the values (Systolic and Diastolic).

### SpO2 and Pulse Rate
*   Insert your finger into the Contec CMS60D probe.
*   The device will automatically begin reading. 
*   The dashboard will update with your oxygen saturation (%) and heart rate (BPM).

## 4. Understanding Notifications and Status

### Device Status Indicator
Inside the application, you can check the status of your devices. The `/device_status` endpoint (or the status panel on the dashboard) provides real-time feedback:
*   **Good Condition**: The device is properly connected, recognized by the system, and ready for measurement.
*   **Disconnected/Error**: The system cannot communicate with the device.

### Failure Notifications
The system provides explicit notifications when a measurement fails:
*   **"Arduino not available"**: Triggered when the system cannot receive data from the temperature or BP sensors via the Arduino.
*   **"No valid data available"**: Triggered if the Contec CMS60D is disconnected during a reading or providing invalid pulse/SpO2 values.
*   **"Didn't receive any reply from Arduino"**: Specific to the Blood Pressure measurement start/stop command.

## 5. Health Status Descriptions
The system analyzes your readings and provides a "Status" description for each parameter:

### Temperature
*   **Hypothermia**: < 35.0°C
*   **Normal**: 35.0°C - 37.2°C
*   **Fever**: 37.3°C - 38.4°C
*   **High Fever**: ≥ 38.5°C

### Blood Pressure (BP)
*   **Normal**: < 120 (SYS) and < 80 (DIA)
*   **Elevated**: 120-129 (SYS) and < 80 (DIA)
*   **Hypertension Stage 1**: 130-139 (SYS) or 80-89 (DIA)
*   **Hypertension Stage 2**: 140-179 (SYS) or 90-119 (DIA)
*   **Hypertensive Crisis**: ≥ 180 (SYS) or ≥ 120 (DIA)

### SpO2 (Oxygen Level)
*   **Normal**: ≥ 95%
*   **Low Oxygen**: 90% - 94%
*   **Critical**: < 90%

### Heart Rate (Pulse)
*   **Bradycardia**: < 60 BPM
*   **Normal**: 60 - 100 BPM
*   **Tachycardia**: > 100 BPM

## 7. Advanced: Knowledge Base (FAISS & RAG)
The system uses Retrieval-Augmented Generation (RAG) to provide more accurate health reports by "consulting" a local knowledge base.

### Ingesting New Knowledge
You can feed new sensor readings, symptom patterns, or medical knowledge into the system using the `ingest_data.py` script.
1.  **Place your data**: Put your `.json` or `.csv` files inside the `/knowledge` folder.
    -   **CSV Format**: Use headers like `reading_type, value, symptoms, analysis, recommendation`. See [knowledge/health_patterns.csv](file:///c:/Users/MARICON/OneDrive%20-%20Polytechnic%20University%20of%20the%20Philippines/Documents/Website/BP-AI-FlaskX/knowledge/health_patterns.csv) for a template.
    -   **JSON Format**: Use a list of objects with a `"text"` field.
2.  **Run the Ingestion**:
    ```powershell
    python ingest_data.py knowledge/your_file.csv
    ```
3.  The script will generate a `faiss_index.bin` and `faiss_meta.json`.

### How it works
When you generate a report, the system automatically searches this index for medical context related to your current vitals and symptoms, injecting relevant info into the AI's prompt for a more "informed" result.

## 8. Troubleshooting
If you encounter a "Disconnected/Error" status or a failure notification:
1.  **Check Physical Connections**: Ensure USB cables are securely plugged in.
2.  **Verify COM Ports**: Check the Windows Device Manager to ensure the Arduino is assigned to the correct port (default is `COM5`).
3.  **Run as Administrator**: Some HID devices (like the Contec oximeter) require administrative privileges to access.
4.  **Restart the App**: If a device was plugged in AFTER the app started, refreshing or restarting may be required.

## 6. Report Generation
After all measurements are taken:
1.  Enter patient details (name, symptoms, etc.).
2.  Click "Generate Report".
3.  The system uses AI to analyze your vitals and produce a detailed PDF report.
4.  Reports are automatically uploaded to Firebase and backed up locally in `local_backup.db`.

## 6. Shutdown
When finished, use the "/shutdown" option in the application menu to safely power down the computer system.
